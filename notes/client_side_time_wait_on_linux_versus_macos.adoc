import socket
from threading import Thread


def client():
    sock = socket.create_connection(("127.0.0.1", 65442))
    sock.close()


srv1 = socket.socket()
srv1.bind(("127.0.0.1", 65442))
srv1.listen()
Thread(target=client).start()
conn, _ = srv1.accept()
while conn.recv(1):
    pass
conn.close()
srv1.close()

srv2 = socket.socket()
srv2.bind(("127.0.0.1", 65442))

error on mac
ok on linux

So the final picture:

client-side TIME_WAIT
tcp4       0      0  localhost.53546        localhost.65442        TIME_WAIT
loopback, so server kernel sees this TIME_WAIT
on linux, bind fails if 65442 in local column
on mac, bind fails if 65442 in local or foreign column
Loopback is necessary for the failure to be possible — without it, the client-side TIME_WAIT lives in a different kernel and the server's bind can't see it
macOS's strict bind check is what turns that visibility into an actual refusal. Linux can see the same TIME_WAIT and shrugs. (My BSD-quirk point.)
Both factors are needed, and the Docker test isolates which one is doing the refusing. Nice empirical resolution.
